# Practice, Practice, Practice

Source: Dave Hoover and Adewale Oshineye, *Apprenticeship Patterns* (2009) — provenance: author-verified

## When it applies

- A skill can be done with thought but not fluidly, and it keeps mattering in real work
- A concept is understood but never embedded
- There's slack time between features worth investing in deliberate skill work

## The rule

Practice is not work: real work has stakes and never lets you try the same problem five ways; a kata removes the stakes so you can. Repeat with variation — same kata, different language, constraint, or order — because identical repetition teaches one solution, not the skill. Time-box to 15–60 minutes, reflect at the end (without reflection it's exercise, not learning). Operational concepts warrant katas; mindset concepts are practiced by living them.

## Anti-patterns

- **"I learn by doing real work"** — exposure without repetition-with-variation isn't deliberate practice
- **Kata as benchmark** — timing and elegance-scoring miss the point; the point is the doing
- **Skipping the reflection** — five katas without reflection are five exercises
- **Practice as production** — kata code is throwaway by design; polishing it for portfolio defeats the stakes-free purpose

## In the agentic era (observed 2026-07)

The practice is the human's — the agent facilitates (prompts, hints, reflects) and never runs the kata in its own head as a substitute; "let me show you how it would go" is the anti-pattern wearing a helpful face. Don't grade or score the human's kata work; reflection prompts ask what the human noticed, not what the AI judges. Suggest a fresh variation each time so repetition stays meaningful.
