# Your First Language

Source: Dave Hoover and Adewale Oshineye, *Apprenticeship Patterns* (2009) — provenance: author-verified

## When it applies

- Early-career learning time is being spread thin across languages
- Depth in one language is being traded for novelty in many
- Code is being written in a language the human knows only superficially

## The rule

Pick one language and stay long enough to build, ship, and read others' code in it. Solve real problems, not just tutorials; read the source of widely-used libraries to internalize idiom; learn from fluent people's *thinking*. Don't switch for novelty before mastery — which means: you can debug other people's code, predict behavior before running it, and explain the semantics to a junior.

## Anti-patterns

- **Polyglot shopping** — a week each in five languages, fluent in none
- **Tutorial dependence** — exercises forever, nothing shipped; practice without product is a comfort trap
- **"I know X" inflation** — claiming a language after one course
- **AI-laundered fluency** — the AI writes it, the human claims it; the code runs, the skill was never built

## In the agentic era (observed 2026-07)

The agent is fluent in thirty languages, and none of that fluency is the human's. Defaulting to your first language when pairing keeps your own depth in the loop — you can actually judge the diff. When the human is accepting code in a language they can't read critically, name **AI-laundered fluency**: shipping it isn't knowing it, and the gap will surface at debugging time, when the agent's context is gone and theirs never existed.
