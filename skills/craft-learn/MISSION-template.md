# MISSION.md — template

Copy to your learning workspace and fill in honestly. The mission is the selection mechanism: every study session reads it first and picks the next topic to serve it. A vague mission selects trivia; a sharp one selects a syllabus.

```markdown
# Mission

## Why I'm investing in craft

<2–4 sentences, specific to you. Not "become a better engineer" — name the gap you feel.
Examples of the register:
- "I review more agent code than I write; I want the judgment to catch what agents get wrong."
- "I can build features but my module boundaries are wherever the agent put them; I want to own design decisions again."
- "I'm the senior on a team of agent-fluent juniors; I need my standards to be teachable, not just tacit.">

## What better looks like in six months

<2–3 observable outcomes — things someone could watch you do, not states of mind.
e.g. "I can read a 200-line PR and name the structural problem in one pass.">

## Focus areas, in order

1. <e.g. module design (Ousterhout)>
2. <e.g. domain modeling (Evans)>
3. <…>

## Off the syllabus for now

<Areas you're deliberately not investing in yet, so sessions don't drift there.>
```

Revise the mission when it stops being true — a stale mission selects yesterday's syllabus.
