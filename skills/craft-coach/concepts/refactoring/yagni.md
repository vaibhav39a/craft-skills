# YAGNI (You Aren't Gonna Need It)

Source: Extreme Programming, coined on the C3 project (Kent Beck to Chet Hendrickson); analysis by Martin Fowler — provenance: web-sourced (https://martinfowler.com/bliki/Yagni.html)

## When it applies

- Code is being written for a feature nobody is shipping yet — a config option, an abstraction layer, a "for future extensibility" parameter
- A design argument leans on a presumed future need
- YAGNI is being invoked to skip tests or refactoring (it doesn't cover those — see below)

## The rule

Don't build a **presumptive feature** — code supporting a capability not yet being made available for use. Fowler's analysis names four costs, sharper than the slogan: **cost of build** (the effort itself), **cost of delay** (the features you didn't ship instead), **cost of carry** (its complexity taxes every future change), and **cost of repair** (the need turns out different than presumed). The part secondhand tellings miss: carry and delay apply *even when the prediction was right*. The fence matters equally: YAGNI does not apply to effort that makes software easier to modify — refactoring, tests, and delivery infrastructure are what make YAGNI viable, not violations of it.

## Anti-patterns

- **The presumptive feature** — speculative capability, speculatively general interfaces, the unused flexibility
- **YAGNI as sloppiness** — invoking it against tests, refactoring, or design care; Fowler explicitly fences this off

## In the agentic era

Speculative generality is the agent reflex: options objects nobody passes, strategy interfaces with one implementation, "extensible" plugin seams for plugins that don't exist — presumptive features generated by default because generality *looks* like quality. The cost of build collapsed, which tempts everyone to shrug — but cost of carry didn't move: every generated abstraction taxes every future change, human and agent alike, and it now accrues at generation speed. When a draft builds for a need nobody stated, name the presumptive feature and ask which shipped requirement wants it. This may be the canon's single highest-frequency surfacing moment.
